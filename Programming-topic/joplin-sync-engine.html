Concept สำคัญ
Conceptคืออะไรใช้ทำอะไร
cursor ตัวเลข integer ที่เพิ่มทีละ 1 ทุกครั้งที่ remote มีการเปลี่ยนแปลง บอก device ว่า "sync ถึงตรงไหนแล้ว" ไม่ต้อง pull ทั้งหมด
sync_state เก็บ hash ของ .md และเวลาที่ sync ล่าสุดสำหรับแต่ละ note ตรวจว่า local note ถูกแก้ไขหลัง sync หรือเปล่า (conflict detection)
delta log ไฟล์ JSON ที่บันทึก action ทุกอย่างบน remote (put/delete) + timestamp ให้ device ดึงเฉพาะ "สิ่งที่เปลี่ยนแปลง" แทน list ทั้งหมด
conflict local แก้ไข note AND remote ก็แก้ note เดียวกันพร้อมกัน บันทึกทั้ง 2 เวอร์ชันใน _conflicts/ ให้ผู้ใช้ตัดสินใจ
deleted_time UNIX ms (เช่น 1772272960230) หรือ 0 ถ้ายังไม่ลบ บอก device อื่นว่า note นี้ถูกลบไปแล้ว
full sync ดึงทุกไฟล์จาก remote ไม่สนใจ cursor ใช้ครั้งแรกเท่านั้น (device ใหม่) หลังจากนั้นใช้ delta sync
